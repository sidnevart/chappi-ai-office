# Мультиагентная работа

Chappi AI Office поддерживает параллельную работу нескольких специализированных агентов.

## Типы агентов

| Агент | Специализация | Когда запускается |
|-------|--------------|------------------|
| Основной агент | Общение, координация | Всегда активен |
| Агент инфраструктуры | Сервер, Docker, развёртывание | При системных задачах |
| Агент исследований | Поиск в интернете, анализ | При исследовательских задачах |
| Агент базы знаний | Сохранение и поиск в БД | При работе с памятью |

## Как запустить субагента

Просто попросите в Telegram:

```
Запусти агента для исследования: найди лучшие практики CI/CD в 2026 году
Попроси инфраструктурного агента проверить состояние сервисов
Агент базы знаний: сохрани всё что мы обсудили сегодня
```

Основной агент сам решает, когда делегировать задачу специализированному субагенту.

## Отображение в Star Office UI

Каждый активный агент отображается как отдельный персонаж в пиксельном офисе на [http://80.74.25.43:3000](http://80.74.25.43:3000).

Когда субагент запускается, он регистрируется в офисе:
- Появляется новый персонаж
- Отображается его имя и текущее состояние
- По завершении задачи персонаж уходит

## Общая база знаний

Все агенты работают с одной и той же базой данных PostgreSQL. Это означает:

- Агент исследований сохраняет отчёт → основной агент его видит
- Основной агент создаёт задачу → агент базы знаний её обновляет
- Инфраструктурный агент логирует действие → в Grafana появляется событие

## Ограничения

- Субагенты являются частью одного процесса OpenClaw, не отдельными серверами
- Параллельность ограничена контекстным окном модели
- Каждый субагент работает последовательно внутри сессии, если не настроен иначе
