# База знаний

Агент хранит информацию в структурированной базе данных PostgreSQL на сервере. Всё что вы просите запомнить — сохраняется навсегда.

## Как сохранять

Просто скажите агенту в Telegram:

```
Запомни: я предпочитаю краткие ответы без лишних объяснений
Сохрани ссылку: https://example.com — документация по API
Создай задачу: разобраться с OAuth до пятницы
Запомни факт: мой часовой пояс — UTC+3
```

Агент сам выберет нужную таблицу и сохранит информацию.

## Таблицы базы данных

| Таблица | Что хранится |
|---------|-------------|
| `notes` | Факты, заметки, личные предпочтения |
| `tasks` | Задачи с приоритетом и статусом |
| `links` | Сохранённые ссылки с описанием |
| `research_reports` | Результаты исследований |
| `profile_memory` | Долгосрочная информация о вас |
| `event_log` | Журнал действий агента |

## Как искать

```
Найди в базе знаний всё про OAuth
Покажи мои задачи на эту неделю
Что ты знаешь про мои предпочтения по моделям?
```

Агент использует семантический поиск через MemPalace и структурированный запрос через PostgreSQL.

## Семантическая память (MemPalace)

Помимо явно сохранённых фактов, агент ведёт семантическую память — автоматически индексирует контекст разговоров. Это позволяет находить связанную информацию даже без точного совпадения слов.

```
Ты помнишь, что мы обсуждали про модели?
Расскажи что знаешь обо мне
```

## Прямой доступ через командную строку

```bash
# Подключение к базе данных
PGPASSWORD=postgres psql -h 80.74.25.43 -p 5432 -U postgres -d ai_office

# Все заметки
SELECT content, tags, created_at FROM notes ORDER BY created_at DESC;

# Незавершённые задачи
SELECT title, priority, due_date FROM tasks
WHERE status != 'done' ORDER BY priority DESC;

# Последние исследования
SELECT title, created_at FROM research_reports ORDER BY created_at DESC LIMIT 5;
```
